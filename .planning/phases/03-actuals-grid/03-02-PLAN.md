Activity-specific derived-field keys + computeFrom (READ CAREFULLY — computeFrom is the change-detection dependency set, NOT the formula):
  - counter-wall actualColumns: status, latitude(lat), longitude(long), wallShopNo, executionDate(date), executionMonth, remarks, actualSqft(number), perUnitCost(currency), totalCost(currency, computeFrom ["actualSqft","perUnitCost"])
      → D3-04: counter-wall has NO derived sqft (uses entered actualSqft). totalCost = actualSqft × perUnitCost.
  - gsb / nlb actualColumns: status, gsbType|nlbType, length(number), breadth(number), height(number), totalSqft(number, computeFrom ["length","breadth","height"]), perUnitCost, totalCost(computeFrom ["totalSqft","perUnitCost"]), remarks
      → D3-04 CRITICAL: totalSqft = length × breadth ONLY. HEIGHT IS EXCLUDED FROM AREA (stored for reference). The computeFrom lists height so the grid refreshes when height changes, but the formula must NOT multiply by height.
  - in-shop actualColumns: status, length, breadth, totalSqft(computeFrom ["length","breadth"]), perUnitCost, totalCost(computeFrom ["totalSqft","perUnitCost"]), remarks
      → D3-04: totalSqft = length × breadth. totalCost = totalSqft × perUnitCost.
  - pop-dealer-kit actualColumns (per-LINE shape): itemName, qty(number), rate(currency), lineTotal(currency, computeFrom ["qty","rate"])
      → D3-04: lineTotal = qty × rate. (POP has NO status field — its executions leave status null.)
  - dealer-certificate actualColumns: status, issuanceDate(date), cost(currency). No derived fields.
