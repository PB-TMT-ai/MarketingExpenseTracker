<objective>
Build the executions persistence slice — the data-access module and the Server Action that saves the grid's batched edits with per-unit optimistic concurrency. This is a near-clone of `commitPlanUpload` (requireSession + Zod + one `db.transaction`) with the FK-restrict catch swapped for a per-unit `version`-check conflict collector. It ships ZERO UI; it is fully provable by PGlite-backed vitest. The single hardest correctness property of the phase after the off-plan guard lives here: D3-11 lost-update prevention.

Purpose: Give the Wave-3 grid a trustworthy save endpoint. The structural off-plan guard (no sfid on executions; NOT NULL plan_row_id FK) and the version check are both enforced here, server-side, regardless of what the browser sends.
Output: `lib/db/executions.ts`, `lib/actions/executions.ts`, a live PGlite smoke for the cross-driver rowCount gate, a vitest suite mirroring `plans.test.ts`, and a `executions:smoke` npm script.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Server Action | The grid posts a batch patch array of untrusted JSON (versions, fields, totals, planRowId) |
| Server Action → Database | Versioned writes + transactional kit writes cross into Postgres |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | Lost-update / silent overwrite on shared login | mitigate | updateExecutionVersioned UPDATE ... WHERE id=? AND version=? + rowCountOf===1; conflict collected (block-not-overwrite), unaffected units still commit (D3-11). Proven by the version-conflict-isolation vitest |
| T-03-05 | Tampering / Elevation | Off-plan spend injection (write execution for unplanned SFID) | mitigate | Structural — executions has NO sfid column; the only write path is planRowId NOT NULL FK to an existing plan_row. Zod schema rejects any sfid field; planRowId must reference a real row or the FK rejects the insert (COMP-01) |
| T-03-06 | Spoofing / Elevation | Middleware bypass (CVE-2025-29927) → unauth grid write | mitigate | requireSession() at the entry of saveExecutionsBatch (jose JWT verify); proxy/middleware is UX only. Proven by the auth-rejected vitest |
| T-03-07 | Tampering / DoS | Forged/oversized patch payload; lying client totals | mitigate | Zod re-validates every unit + caps units array length; server re-runs computeDerived for non-overridden derived cells (never trusts client totals). Proven by zod-rejected + server-recompute-trust vitests |
| T-03-08 | Tampering | SQL injection via field values | mitigate | Drizzle parameterized queries throughout; no string SQL with user input |
| T-03-09 | Tampering | Partial-write corruption mid-batch | mitigate | Single db.transaction; genuine failures roll back; only per-unit version conflicts are collected per D3-11 |
| T-03-SC | Tampering | (no new package installs in this plan) | n/a | Package legitimacy handled in 03-01; this plan adds no dependencies |
