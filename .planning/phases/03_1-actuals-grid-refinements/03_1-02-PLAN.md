<objective>
Deliver GRID-09: targeted hot-path performance fixes in `actuals-grid.tsx`, with NO rewrite, NO TanStack, NO row-store/server-side row model, NO custom canvas (D3.1-05 — Community AG Grid stays).

This plan replaces the per-keystroke clone-on-edit state model with AG Grid's transaction API and React 19 deferral:
- `rowsRef` (off-React Map) as the authoritative row store; `dirtyKeys` Set in state (drives only the save-bar count).
- `handleCellValueChanged` mutates the row object and calls `api.applyTransaction({ update: [row] })` — refreshes ONLY that row's node (getRowId is already set, R10), eliminating the `setRowMap(new Map(prev))` O(N) clone + `rowData` array rebuild + full `setRowData` reconcile on every keystroke.
- `useDeferredValue(sfidSearch)` debounces the external filter re-run (only the typed search needs deferral; facet clicks are discrete).
- `singleClickEdit` grid option so the most-edited cell (status) opens on one click; `animateRows={false}` for a data-entry grid.

Purpose: This refactor REDEFINES the state model (`rowsRef` + `dirtyKeys`) that Plan 04 (GRID-12 save bar + GRID-13 paste) and Plan 05 (COMP-04 off-plan button) all build on. It MUST land before any other plan edits `actuals-grid.tsx`. It runs in Wave 1 parallel with Plan 01 because they share zero files.

Output: Refactored `actuals-grid.tsx` (D3-05 sticky override + D3-11 conflict handling + getRowId all preserved); `baseline-perf.md` with before/after numbers; a human-verify checkpoint confirming the grid still edits/saves correctly and feels faster.
</objective>
