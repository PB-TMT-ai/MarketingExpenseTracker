<objective>
Build the **read-only Drizzle aggregate query layer** that the Phase 4 dashboard page (Plan 04-03) and the geo-drill-tree client island (Plan 04-04) consume. Every dashboard card, chart, and tree row is fed by exactly one helper in this file.

Six helpers, one file:

1. `aggregateScopeTotals(filters)` — single-row aggregate feeding the StatStrip (DASH-01, DASH-03, DASH-05).
2. `aggregateByActivity(filters)` — grouped by `plan_rows.activity` (DASH-02 by-activity card).
3. `aggregateByRegion(filters)` — grouped by `plan_rows.region` (DASH-02 by-region card).
4. `aggregateByGeo(filters)` — flat (region, state, district, taluka) aggregate (DASH-07 drill tree feed).
5. `aggregateWeeklyBuckets(filters, mode)` — `date_trunc('week', executionDate)` bucketed with mode = `{ kind: 'period' }` (between period start/end) or `{ kind: 'rolling', weeks: 4|8|12 }` (last N ISO weeks ending today) (DASH-06).
6. `aggregateExceptionTotals(filters)` — the ONLY helper that selects `source = 'exception'` (D-07).

Purpose: every Phase 4 number renders through one of these six helpers — no inline SQL in pages or components, no JS-side aggregation across full row sets (CLAUDE.md invariant). Numeric boundary-coerced at the helper exit (Pitfall 8). Exception exclusion (D-06) and cancelled-cost exclusion (Pitfall 2) baked in at the SQL level, defended by integration tests.

Output: `lib/db/dashboard.ts` + `lib/db/dashboard.test.ts`.
</objective>
