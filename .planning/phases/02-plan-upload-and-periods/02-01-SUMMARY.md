## Accomplishments
- **SheetJS CE 0.20.3 installed from CDN tarball** — `xlsx: "https://cdn.sheetjs.com/xlsx-0.20.3/xlsx-0.20.3.tgz"` in package.json (no caret range). CVE-2023-30533 (prototype-pollution on read) is structurally excluded by the install path + the verify gate against `"^"` ranges.
- **`lib/excel/types.ts`** — Five canonical types (Classification, ParsedRow, HeaderError, FieldError, PreviewRow) the entire upload pipeline consumes. Zero runtime code by design.
- **`lib/excel/parse.ts`** — `readWorkbook(buf)` uses `{ type: "array", cellDates: true }` + `sheet_to_json({ header: 1, raw: false, defval: "", blankrows: false })` exactly per RESEARCH §1. `coerceCell(raw, FieldDef)` handles all eight FieldKinds with per-kind logic: text/status/enum/lat/long preserve digits without scientific-notation loss; number/currency strip `/[₹$,\s]/g`; date accepts JS Date (UTC-formatted via `getUTCFullYear/getUTCMonth/getUTCDate`) and DD/MM(/YY|YYYY) strings, but REJECTS ISO YYYY-MM-DD on purpose.
- **`lib/excel/template.ts`** — `buildPlanTemplate(key)` round-trips through `readWorkbook` and emits headers === `getActivity(key).planColumns.map(c => c.label)` in order for all six activities. `TEMPLATE_FILE_NAME(key)` returns the D2-05 convention.
- **`lib/excel/validate.ts`** — `validateHeaders` enforces D2-03 (case-insensitive + whitespace-trim, ordered) and returns one of three HeaderError kinds (missing → extra → mismatch). `buildPreview` emits PreviewRows with Excel-1-indexed rowNumber, routes per FieldDef.shared (real columns vs jsonb tail), assigns the three browser-local classifications, and runs a second pass to flip both occurrences of any duplicate SFID (fieldError wins over duplicate).
- **`lib/excel/util.ts`** — `chunked<T>` generator for 500-row bulk-insert batches that Plan 02-02's transaction will consume.
- **SECURITY header** at the top of `parse.ts` so a future contributor can't quietly add a Route Handler that parses `.xlsx` server-side.

## Decisions Made
See `key-decisions` frontmatter. Headlines:
1. **CDN tarball verbatim, no caret range** — re-installs must re-fetch the audited 0.20.3 bytes.
2. **ISO date is REJECTED, not silently accepted** — surfaces vendor template drift.

5. **`fieldError` wins over `duplicate`** — user must fix the parse before duplicate even matters.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Defensive zero-sheets test in parse.test.ts was unprovable via SheetJS API**

- **Found during:** Task 2 GREEN run
- **Issue:** The behavior bullet "readWorkbook throws when the workbook has zero sheets" couldn't be proven via `XLSX.write(book_new())` because SheetJS refuses to write a zero-sheet workbook (validates at write time with `Workbook is empty`). I also tried `vi.spyOn(XLSX, "read")` but vitest 4 + ESM module namespace refuses property reconfiguration.
- **Fix:** Extracted the sheet-selection logic into an internal `_extractFirstSheetRows(wb)` helper that takes a hand-built `{ SheetNames: [], Sheets: {} }`. Exported with underscore prefix to mark it not-public-API. The zero-sheets defensive guard is now unit-testable via two tests (empty SheetNames AND SheetNames[0] points at missing sheet). `readWorkbook` still calls `_extractFirstSheetRows` internally so production behavior is unchanged.
- **Files modified:** `lib/excel/parse.ts`, `lib/excel/parse.test.ts`
- **Commit:** `3ec8f46` (Task 2 GREEN)
