<source_of_truth>
The SIX activities and their EXACT columns are enumerated in `.planning/PROJECT.md` → "Context" → "Activity column specs" (lines ~50-55). Use them VERBATIM — do not re-derive or simplify. Reproduced here for the executor:

- **Counter Wall Painting** — type `measurement`.
  Plan columns: Region, SFID, Dealer/Area, State, District, Taluka, Plan Sq Ft, Distributor.
  Actual columns: Status, Latitude, Longitude, Wall/Shop No (label note: VendorInitials_wallNo_DD/MM/YY), Execution Date, Execution Month, Remarks, Actual Sq Ft, Per-unit cost, Total cost.
  Computed: Total cost (Actual Sq Ft × Per-unit cost) — mark `computeFrom`.

- **GSB** and **NLB** — type `measurement` (two SEPARATE activities sharing the same column shape).
  Plan columns: Region, SFID, Dealer Name, State, District, Taluka, Distributor.
  Actual columns: Status, GSB/NLB type, Length, Breadth, Height, Total Sq Ft, Per-unit cost, Total cost, Remarks.
  Computed: Total Sq Ft (from Length×Breadth×Height), Total cost (Total Sq Ft × Per-unit cost) — mark `computeFrom`.

- **In-shop Branding** — type `measurement`.
  Plan columns: Region, SFID, Dealer Name, State, District, Distributor, Pin code, GST No., Mobile No.
  Actual columns: Status, Length, Breadth, Total Sq Ft, Per-unit cost, Total cost, Remarks.
  Computed: Total Sq Ft (Length×Breadth), Total cost (Total Sq Ft × Per-unit cost) — mark `computeFrom`.

- **POP / Dealer Kit** — type `item-list`.
  Plan (parent) columns: Region, State, District, Distributor, SFID, Dealer.
  Actual columns: recorded as line items (item name, qty, per-unit cost, line total) — item names come from the managed item master (ACTV-04). Model the line-item shape via the `item-list` type; the per-line FieldDefs are item, qty, rate, total.

- **Dealer Certificate** — type `status`.
  Actual columns: issuance Status, issuance Date, Cost. Plan columns: the shared who/where set (Region, State, District, Distributor, SFID, Dealer).
</source_of_truth>

<shared_columns_note>
The who/where set — Region, State, District, Taluka, Distributor, Dealer, SFID — maps to REAL indexed columns on `plan_rows` (see lib/db/schema.ts: region/state/district/taluka/distributor/dealer/sfid). Mark these FieldDefs `shared: true`. Activity-specific fields (measurements, lat/long, remarks) are NOT shared (they live in the jsonb `fields` tail). This `shared` flag is what Phase-2 import uses to route a column to a real column vs the jsonb tail — declare it now, do not implement routing here.
</shared_columns_note>
